Klantenportaal — Datamodel & Unified Chat

Schema-wijzigingen en hoe klant-chat in het dashboard komt

Klant ↔ User koppeling

Nieuw veld in klanten tabel
// convex/schema.ts → klanten tabel
klanten: {
userId: id("users"),      // eigenaar (bedrijf)
naam, adres, email, ...  // bestaande velden
clerkUserId?: string,    // NIEUW: Clerk account van de klant zelf
portalEnabled?: boolean,  // NIEUW: heeft portaal-toegang
lastLoginAt?: number,    // NIEUW: laatste portaal-login
}
Flow: Medewerker maakt klant aan → klikt "Portaal activeren" → systeem stuurt uitnodigings-email → klant registreert bij Clerk → clerkUserId wordt gekoppeld aan klanten record

Unified Chat — Nieuw: chat_threads tabel

Nieuw schema: chat_threads
// Vervangt offerteMessages met een flexibeler systeem
chat_threads: {
type: "klant" | "team" | "project",
klantId?: id("klanten"),      // voor klant-threads
offerteId?: id("offertes"),   // context: welke offerte
projectId?: id("projecten"),  // context: welk project
participants: string[],      // Clerk userIds
lastMessageAt?: number,
lastMessagePreview?: string,
unreadByBedrijf?: number,
unreadByKlant?: number,
userId: id("users"),        // bedrijf-eigenaar
}
Nieuw schema: chat_messages (unified)
chat_messages: {
threadId: id("chat_threads"),
senderType: "bedrijf" | "klant" | "medewerker",
senderUserId: string,       // Clerk userId
senderName: string,         // display name
message: string,
attachments?: string[],      // file URLs
isRead: boolean,
createdAt: number,
}
